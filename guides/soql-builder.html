🔍 SOQL Builder Guide

What It Does

The SOQL Builder is a full-featured query tool for Salesforce Object Query Language. Build queries visually with a guided form, write them directly in the editor, and execute them against your org. Results appear in a high-performance virtualized table that smoothly handles 1000+ rows and 50+ columns.

Edit records inline, navigate relationships with clickable links, filter columns, search globally, export results, and pop queries out to a separate window for side-by-side comparison.

How to Access

Open the TrackForcePro workspace and click the SOQL tab. The object selector loads your org's SObjects automatically. You're ready to start building or editing queries.

Object Selector

The object selector is a searchable combobox — not a native dropdown. Start typing an object name and the list filters in real time with substring highlighting.

  • Type to Search — Begin typing "Account" or "Case" and matching objects appear instantly.
  • Arrow Keys — Use Up and Down arrows to navigate the dropdown list.
  • Enter to Select — Press Enter to pick the highlighted object.
  • Escape to Close — Press Escape to close the dropdown without selecting.

Selecting an object populates the query builder's FROM clause and triggers field suggestions.

Query Builder

The visual builder has sections for each part of your SOQL query:

  • SELECT Fields — Click to add field names. The builder suggests available fields as you type.
  • FROM Object — Set via the object selector above. Only change this to switch objects.
  • WHERE Clauses — Add filter conditions. The builder guides you with field names, operators (=, !=, >, <, LIKE, IN, etc.), and value inputs.
  • ORDER BY — Sort results by field name in ASC or DESC order.
  • LIMIT — Cap the number of rows returned (max 2000 per API limit).
  • OFFSET — Skip the first N rows (useful for pagination).

The suggestion engine is clause-aware — it knows whether you're building SELECT, WHERE, or ORDER BY and suggests context-appropriate fields.

Query Editor

Write or paste SOQL directly in the full text editor. The editor supports:

  • Syntax highlighting for SOQL keywords.
  • Bracket matching and auto-indentation.
  • Line numbers for easy reference.
  • Collapse/expand toggle to give more room for results after execution.

Toggle between visual builder mode and text editor mode. You can switch back and forth — the builder will parse what you've written.

Executing Queries

Once your query is ready:

  • Click the Run button, or
  • Press Ctrl+Enter (Windows/Linux) or Cmd+Enter (Mac).

Results appear in the virtualized table below. You can collapse the editor after execution to maximize vertical space for results.

Result Table

Results render in a virtualized table (SoqlVirtualTable) that shows only visible rows, rendering smoothly even with 1000+ rows and 50+ columns.

Density Modes

Choose your preferred row height from the Density dropdown:

  • Normal — 38px rows, comfortable spacing (default).
  • Compact — 26px rows, efficient for scanning.
  • Dense — 20px rows, maximum data density.

Column Features

  • Sticky Header — The header row stays visible as you scroll.
  • Sticky First Column — The first (usually ID) column pinned on the left while scrolling right.
  • Column Pinning — Pin up to 3 columns to keep them visible. Right-click a column header to pin.
  • Column Resizing — Drag the column header border to resize.
  • Column Sorting — Click a column header to sort ascending/descending. Visual indicator shows sort direction.
  • Per-Column Filtering — Click the Filter toggle to show input fields under each column. Type to filter that column's values.

Search & Selection

  • Global Search — Search across all visible cells. Matching text is highlighted in the table.
  • Row Selection — Checkboxes in the first column select rows for bulk actions.
  • Keyboard Navigation — Arrow keys move between cells. Enter selects a row. Escape deselects.

Smart View Mode

Toggle Smart View to automatically hide empty and null columns. Great for wide result sets with sparse data.

Inline Editing

If inline editing is enabled for your org (Settings > General > Org Management), you can edit records directly in the result table:

  • Enable Inline Edit — Toggle the Inline Edit button in the toolbar.
  • Double-Click to Edit — Double-click any updateable cell to enter edit mode.
  • Type-Aware Inputs — Text fields are text inputs, boolean fields are dropdowns (True/False), large text fields are textareas.
  • Save or Cancel — Press Enter to save (PATCH to REST API), or Escape to cancel.

Inline edits are disabled per-org in Settings if your org requires it (e.g., production safety). A warning notifies you if the feature is blocked.

Relationship Field Links

When your SOQL query includes dot-notation relationship fields (e.g., Account.Name, Owner.Email), they render as clickable links in the result table.

  • Click to Navigate — Click a relationship link to load that related record into the Data Explorer.
  • Copy Field Value — Right-click the link to copy the field's value to your clipboard.
  • Breadcrumb Trail — Navigation automatically builds a breadcrumb trail so you can backtrack to previous records.

Pop-Out Results

Click the Pop-Out button in the toolbar to open your results in a separate browser window. The pop-out window has:

  • Full SoqlVirtualTable feature parity (density, sorting, filtering, search, column pinning).
  • Full SoqlResultToolbar (all controls available).
  • Independent state — you can modify columns or run a new query in the main tab without affecting the pop-out.

Perfect for side-by-side comparison, presenting results to stakeholders, or reference while building a follow-up query.

Tips & Best Practices

  • Smart View for Wide Results — Toggle Smart View to hide empty columns in result sets with 30+ columns.
  • Pop-Out for Comparison — Pop-out your first query result, then run a second query in the main tab. Compare side-by-side without switching windows.
  • Test with LIMIT — Always test large queries with a LIMIT clause first (e.g., LIMIT 10) to verify syntax before running against thousands of rows.
  • Suggestion Engine Reset — The suggestion engine resets between queries. If suggestions feel stale, close and reopen the SOQL tab.
  • Export for Reporting — Export filtered results as CSV to import into Excel, Google Sheets, or your reporting tool.
  • Keyboard Shortcuts — Press Ctrl+Enter to run, arrow keys to navigate table, Enter to select rows, Escape to deselect.
  • Column Pinning — Pin the ID column plus 2 data columns to keep key info visible while scrolling right.
💡 Pro Tip: For performance with very large result sets (10,000+ rows), use Compact or Dense density mode to render faster. The virtualized table only renders visible rows, but denser rows = fewer scroll events = smoother performance.

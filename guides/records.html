🔬 Records Guide Workspace Tab

Inspect Salesforce records in depth with all field values, metadata, sharing rules, field history, and user permissions.

What It Does

The Records tab is your deep-dive inspector for Salesforce records. View every field value, type, and updateability status. Check field history, sharing rules, and user permissions without leaving TrackForcePro. Sub-tabs for User Manager and Security Manager provide specialized inspections. Features include pinned records for quick re-access, breadcrumb navigation for lookup traversal, rate-limited bulk scanning, and inline editing with per-org permission controls.

How to Access

Open the TrackForcePro workspace popup and click the Records tab (🔬 icon) in the main tab bar. You can also jump directly to Records from:

  • The Command Palette (Ctrl+K / Cmd+K) — search for "Records" or type a record ID
  • The Sidebar — click "Records" shortcut on any Salesforce page
  • Right-click context menu on Salesforce record pages — "Show All Data"

Record Lookup

Enter a Salesforce record ID (15 or 18 characters) into the lookup field at the top of the Records tab. Press Enter or click the load button to fetch the record.

Tip: The describe cache is org-scoped (keyed by orgId and objectName) to prevent stale metadata from leaking across different Salesforce orgs. Record history stores your last 25 lookups per session for quick re-access.

All Fields Table

Once a record is loaded, the All Fields table shows every field on that record object. The table is paginated with 100 fields per page to keep performance smooth even on objects with hundreds of fields.

Column What It Shows
Field Label Human-friendly field name as shown in Salesforce
API Name Exact API field name used in code and queries
Value Current field value for this record
Type Salesforce field type (Text, Checkbox, Lookup, etc.)
Updateable Whether the field can be edited by your user

Use Prev / Next buttons at the bottom to navigate pages, and a page counter shows your position (e.g., "Page 2 of 8").

Inline Editing

Double-click any updateable field (where the "Updateable" column shows true) to edit it inline. An input field appears with the appropriate type:

  • Text fields — Standard `<input type="text">`
  • Checkbox fields — Dropdown with true/false options
  • Long text fields — Multi-line `<textarea>`

To save: Press Enter. To cancel: Press Escape.

Once saved, the edit is sent to Salesforce via PATCH to the REST API endpoint `/sobjects/{Object}/{Id}`. The field updates in real time on the page and also in Salesforce.

Per-Org Permission Guard: Inline editing respects per-org enable/disable settings in Settings > Inline Edit Permissions. Orgs are allowed by default (fail-open). Explicit deny is required to block editing on an org (useful for production).

Pinned Records

Pin frequently-accessed records to avoid re-typing their IDs. Click the Pin button next to any loaded record to add it to your pinned list.

Pinned records are displayed in a #pinned-records-list container at the top of the Records tab. Click any pinned record to reload it instantly. Pinned records are stored per-org via tfp_pinned_records_{orgId} storage key, so each org has its own pinned list.

Breadcrumb Navigation

When you click any lookup field value in the All Fields table, Records automatically navigates to that related record and adds a breadcrumb to the trail. A breadcrumb bar at the top of the tab shows your navigation path.

Click any breadcrumb link to jump back to a previous record. The trail truncates automatically when you navigate backward, keeping your context clean.

Example: You're viewing an Opportunity (001xx). You click the Account lookup field → Records loads the Account (001yy) and shows breadcrumb [Opportunity] > [Account]. Click the Account breadcrumb to return to the Opportunity.

Bulk Record Scan

Scan up to 200 record IDs at once to view multiple records' metadata and values side-by-side. Enter comma-separated record IDs (15 or 18 chars) into the bulk scan field and click Scan.

Processing happens in batches of 5 requests with rate limiting via Utils.createRateLimiter(5) to stay within Salesforce API limits. A progress indicator shows how many records have been scanned and how many remain.

Rate Limiting: Bulk scans are capped at 200 IDs per scan and process 5 at a time to avoid hitting org API call limits. Typical scan of 50 IDs takes 10–15 seconds.

Sub-Tab: User Manager

The User Manager sub-tab (within the Records tab) lets you inspect and manage user records in detail:

  • Search and filter users by name, email, or username
  • View user profile assignments and permission sets
  • Check user activation status, last login, and license type
  • Inline edit user fields (where permissions allow)

User Manager uses the same underlying Records engine, so all standard features (breadcrumbs, pinning, history) work with user records.

Sub-Tab: Security Manager

The Security Manager sub-tab provides specialized inspection of record-level security:

  • Sharing Rules — View all sharing rules that apply to the current record's object
  • Field-Level Security — Check which fields your profile can read and update
  • Object Permissions — Verify your access (Create, Read, Update, Delete) to the record's object

Security Manager helps you debug permission issues without visiting Setup or relying on an admin.

Tips & Best Practices

  • Pin records you check frequently — Avoid re-typing the same IDs repeatedly. Once pinned, a single click loads the record.
  • Use breadcrumbs to explore without losing context — Navigate through related records (Account → Contacts → Opportunities) and jump back with one click.
  • Bulk scan is rate-limited for your safety — 5 concurrent requests prevent API limit overages. Typical 50-record scan takes 10–15 seconds.
  • Inline edit is fail-open by default — All orgs allow editing unless you explicitly disable it in Settings > Inline Edit Permissions. Good for dev/sandbox, bad for production.
  • Block inline editing on production — Use Settings to disable editing on your prod org, preventing accidental data changes by your team.
  • Record history tracks your last 25 lookups — Use the history dropdown to quickly jump to recently-viewed records without re-scanning.

⚙️ Settings Guide Workspace Tab

Configure TrackForcePro appearance, keyboard shortcuts, inline editing permissions, sidebar layout, and org management with centralized settings.

What It Does

The Settings tab is command central for all TrackForcePro configuration. Customize appearance (theme), define keyboard shortcuts, manage per-org inline edit permissions, show/hide sidebar shortcuts, position floating buttons, and manage saved orgs. Changes are applied via the Save & Apply button, which broadcasts settings to all connected Salesforce tabs and refreshes their UI in real time.

How to Access

Open the TrackForcePro workspace popup and click the Settings tab (⚙️ icon) in the main tab bar. Settings are organized into collapsible sections for easy navigation.

Appearance Settings

Control TrackForcePro's visual theme:

  • Light Mode — Bright backgrounds with dark text. Good for bright environments.
  • Dark Mode — Dark backgrounds with light text. Reduces eye strain in dim environments.
  • System — Follow your operating system's theme preference (Windows, Mac, or Linux).

Theme changes apply immediately across all TrackForcePro surfaces (popup, sidebar, overlays). Select your preference and click Save & Apply to persist the choice.

OS Integration: If you choose System theme, TrackForcePro watches your OS settings. Switch your OS to dark mode at night and TrackForcePro automatically follows.

Keyboard Shortcuts

Customize global shortcut actions to match your workflow. Only the sidebar toggle has a built-in default — all others must be assigned manually:

Action What It Does Default
Toggle Sidebar Show/hide the floating sidebar on Salesforce pages Ctrl+Shift+S (Win) / Cmd+Shift+S (Mac)
Toggle Show All Data Open/close the All Fields inspector on record pages Not assigned by default
Command Palette (Page) Open the command palette on Salesforce pages Not assigned by default
Command Palette (Workspace) Open the command palette inside the extension popup Ctrl+K (Win) / Cmd+K (Mac)
Open Setup Search Open Salesforce Setup search on page Not assigned by default
Toggle LWC Explorer Open/close the LWC Explorer overlay Not assigned by default
Open SOQL Builder Open the SOQL Builder from Salesforce pages Not assigned by default
Open GraphQL Builder Open the GraphQL Builder from Salesforce pages Not assigned by default

How to Record a Shortcut

  1. Click the shortcut record button next to the action you want to customize
  2. Press your desired key combination (e.g., Ctrl+Alt+S or Cmd+Option+1)
  3. TrackForcePro records the combination and closes the record dialog
  4. Click Save & Apply to activate the new shortcut

Reset to Defaults: Click the Reset button next to any shortcut to restore its default key combination.

Platform-Aware Display: Shortcuts are shown with platform-specific notation. On Mac, you'll see "Cmd" instead of "Ctrl". On Windows, you'll see "Ctrl". Makes it easy to match your OS shortcuts.

Inline Edit Permissions

Control which orgs allow inline editing (double-click to edit fields). This is crucial for preventing accidental data changes on production orgs.

How It Works

A per-org toggle list shows all known Salesforce orgs you've accessed. Each org has an "Allow Editing" toggle:

  • Toggle ON (default) — Inline editing is allowed. You can double-click to edit fields.
  • Toggle OFF — Inline editing is blocked. Double-click is disabled and edit buttons are hidden.

Orgs are fail-open by default — if an org is not explicitly in the disable list, editing is allowed. This is intentional to avoid surprise blocks.

Permission Guard Locations: The inline edit permission guard is enforced at:
  • Records tab (Data Explorer) — Double-click disabled if blocked
  • SOQL Builder — Inline edit toggle hidden if blocked
  • Show All Data overlay — Double-click disabled if blocked

Migration from Old Setting

If you previously used the old tfpDisableInlineEditProd boolean setting, it automatically migrates to the new per-org list on first Settings load. No manual action needed.

Sidebar Configuration

Show or hide individual Sidebar shortcuts to keep your sidebar clean and focused:

  • Record Tools — Shortcuts for Records sub-tabs (User Manager, Security Manager)
  • Org Tools — Shortcuts for Org Tools panels (Custom Labels, Limits, Metadata Explorer, etc.)
  • Miscellaneous — Other shortcuts like Command Palette, Settings

Toggle switches next to each shortcut control visibility. Hidden shortcuts won't appear in the floating sidebar on Salesforce pages, but you can still access them from the Workspace tabs or Command Palette.

Button Positioning

Customize where the floating buttons appear on Salesforce pages. Three buttons can be positioned independently:

Button What It Does Position Options
Sidebar Button Toggle the floating sidebar Right, Left, Bottom
Show All Data Open the All Fields inspector Right, Left, Bottom
Command Palette Open the command palette Right, Left, Bottom

Bottom Position: When set to Bottom, buttons render as horizontal tabs anchored to the bottom edge of the page. Useful if you prefer a taskbar-style layout.

Independent Positioning: Each button has its own position setting. You could have the Sidebar on the Right, Show All Data on the Left, and Command Palette on the Bottom simultaneously.

Org Management

View and manage all Salesforce orgs you've connected to TrackForcePro. The Org Management section shows a table of saved orgs with:

  • Org Name — Display name or org ID
  • Environment — Production, Sandbox, Scratch, etc.
  • Instance — Salesforce instance (na0.salesforce.com, etc.)
  • Allow Editing — Toggle for inline edit permission (synced with Inline Edit Permissions section)
  • Delete — Button to remove the org record

The org list is populated automatically as you access different Salesforce orgs. Toggle the Allow Editing switch to enable/disable inline editing for each org. Changes sync bidirectionally with the Inline Edit Permissions section.

Org Detection: Orgs are identified by their instance URL. TrackForcePro automatically detects and catalogs new orgs as you log in.

Save & Apply Button

All changes in Settings are applied via the ✅ Save & Apply button at the bottom of the Settings tab.

What It Does

  1. Saves all settings to chrome.storage.local
  2. Applies appearance settings (theme) immediately in the popup
  3. Broadcasts TFP_SETTINGS_CHANGED message to all connected Salesforce tabs
  4. Dispatches per-tab refresh events so each tab re-applies the new settings

Button States

State Appearance Meaning
Neutral Light gray No unsaved changes
Enabled Blue (clickable) Unsaved changes detected
Applying Blue with spinner Settings are being saved and applied
Success Green checkmark Settings applied successfully

The button returns to Neutral after a brief success flash. If there are additional unsaved changes, it becomes Enabled again.

Tips & Best Practices

  • Push settings to all tabs at once — Use Save & Apply to broadcast settings to every open Salesforce tab simultaneously. No need to reload each tab individually.
  • Block inline editing on production — As soon as you add your production org, toggle its Allow Editing switch to OFF in Org Management. Prevents accidental field edits by your team.
  • Customize keyboard shortcuts to your muscle memory — If you're used to different shortcuts from other tools, change them here. Muscle memory is powerful — don't fight it.
  • Use Dark Mode to reduce eye strain — If you're in a dim environment or work late, enable Dark Mode. Choose System to auto-switch based on your OS settings.
  • Hide sidebar shortcuts you don't use — Keep the sidebar lean by hiding shortcuts for features you rarely access. Less clutter = faster scanning.
  • Position buttons where they don't block your workflow — If the right edge is where you usually click, move buttons to the Left or Bottom to avoid accidental activation.
  • Check Org Management regularly — As you access new orgs, they're cataloged automatically. Review the list periodically and adjust inline edit permissions for new production orgs.

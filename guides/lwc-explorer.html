🔎 LWC Explorer

Inspect Lightning Web Components on any Salesforce page — trace server calls, monitor LMS channels, and debug component behavior without browser DevTools.

What It Does

LWC Explorer is an in-page overlay that gives you deep insight into Lightning Web Components and Aura components running on any Salesforce page. Without opening browser DevTools, you can:

  • Inspect components — View the hierarchical component tree, see parent-child relationships, and search by name or type
  • Trace network traffic — Automatically capture server calls (Apex, UI API, GraphQL, REST) associated with each component
  • Debug performance — Review request/response payloads, timing, and error messages inline
  • Monitor LMS channels — Discover Lightning Message Service channels, subscribe/unsubscribe, and publish test messages
  • Navigate records — Click lookup fields in trace details to jump to related records with breadcrumb trails
Key insight: LWC Explorer captures real network traffic from the Salesforce API layer, not mock data. Every trace you see is an actual server call your component made.

How to Open

You can open the LWC Explorer in three ways:

Method Steps
Sidebar Button Click the TrackForcePro sidebar icon in the top-right corner of your Salesforce page, then select LWC Explorer from the menu
Context Menu Right-click anywhere on a Salesforce page and select LWC Explorer from the TrackForcePro context menu
Command Palette Press Ctrl+K (Windows/Linux) or Cmd+K (Mac) to open the Command Palette, then type LWC Explorer and press Enter
💡 Tip: The Command Palette is the fastest way if you set a custom keyboard shortcut. Open Settings → General → Keyboard Shortcuts and customize the LWC Explorer trigger.

Header Controls

The LWC Explorer header provides quick access to scanning, filtering, and recording tools. Here's what each control does:

Button Icon Purpose
Pick Component 🎯 Click, then hover over the page to visually select a component. The selected component is highlighted and scrolled into view in the tree pane.
Refresh Components ⟲ Re-scans the page DOM to rebuild the component tree. Useful after the page loads new components dynamically.
Auto-Capture ⚫ (pulses green when active) Toggle automatic network trace recording. When active, traces are captured every 3 seconds. Click again to stop.
Clear All 🗑️ Removes all captured network traces from the trace panel. Useful for resetting when you want to focus on new activity.
Expand All 📂 (open) Expands all nodes in the component tree to show the full hierarchy at once.
Collapse All 📁 (closed) Collapses all nodes except root-level components. Keeps the tree readable while preserving your ability to drill down.

Component Tree

The left panel displays a hierarchical tree of all LWC and Aura components on the page. At the top are filter pills and a search box.

Filter Pills

Click any pill to toggle its filter:

  • All — Shows all component types
  • LWC — Shows only Lightning Web Components
  • Aura — Shows only Aura components
  • LMS (Beta) — Shows Lightning Message Service channels if discovered

Search Box

Type a component name to filter the tree in real-time. Matching components stay expanded so you can navigate to them quickly.

Selecting a Component

Click any component in the tree to select it. The right panel updates to show the component's metadata (if available) and all related network traces.

Component info: When a component is selected, you'll see its name, a Copy icon to copy it to the clipboard, and a Highlight button to visually highlight it on the page.

Network Traces

The right panel shows all captured network traces (API calls) associated with the selected component. Traces are organized by type and are filterable by category.

Trace Filter Chips

Click any chip to toggle its filter:

  • All — Shows all captured traces
  • Apex — Shows only server-side Apex calls (imperative, wire, enqueued)
  • UI API — Shows Salesforce UI API calls for record data, layouts, and metadata
  • GraphQL — Shows Salesforce GraphQL operations
  • Fetch/API — Shows other REST API calls (tooling, composite, custom)
  • Errors — Shows only traces that returned an error status

Reading a Trace Row

Each trace row displays:

  • Badge — Type indicator (color-coded: blue for Apex, teal for UI API, etc.)
  • Label — Friendly description (e.g., "fetchUserById", "Get Record Layout")
  • Method/Endpoint — HTTP method + target endpoint or Apex class
  • Status — HTTP status code (green 2xx, yellow 3xx, red 4xx/5xx)
  • Timing — Request duration (e.g., "45ms")

Click any trace row to expand the detail pane below and see the full request/response payload.

Trace Categories Explained

Understanding what each trace type represents helps you debug faster:

Category What It Shows Example
Apex Server-side Apex calls triggered by the component. Includes imperative calls, wire method calls, and enqueued actions. Shows the Apex class name and method that processed the call. getUserAccounts from UserDataController
UI API Salesforce UI API calls for record data, field layouts, picklist values, metadata, and more. These power the Lightning UI. GET /ui-api/records/001xx000003DHP
GraphQL Salesforce GraphQL operations. Shows the operation name, query text (if available), variables, and response data or errors. Query getAccounts returning 15 records
Fetch/API Any other REST API traffic: tooling API, custom endpoints, composite API, external integrations, etc. POST /services/data/v59.0/tooling/query
Errors Any trace that returned an HTTP 4xx or 5xx status, or contained an error payload. Helpful for troubleshooting failed operations. 401 Unauthorized, 404 Not Found, 500 Server Error
Internal traffic filtered out: LWC Explorer ignores internal Salesforce service-component Aura traffic (like fingerprinting and telemetry) so you focus only on meaningful business calls.

Trace Details Pane

When you click a trace row, the detail pane expands to show the complete information about that call.

Header Row

  • Call Type — The category (Apex, UI API, GraphQL, etc.)
  • Triggered By — The component name that initiated the call
  • Server Class — (For Apex) The controller class that processed the call
  • Method Name — (For Apex) The method that was invoked
  • When — Timestamp when the call was made (in your browser's local time)
  • Performance — Elapsed time from request to response

Request Section

Shows the HTTP method, endpoint, and request body (if applicable). The body is formatted as readable JSON with syntax highlighting.

Response Section

Shows the HTTP status code and response payload. If the response is JSON, it's formatted and syntax-highlighted for easy scanning. If the response contains an error, it's highlighted and summarized.

Lookup Navigation

If the trace response contains Salesforce record IDs, those IDs appear as clickable links. Click a link to jump to that record in a new panel with a breadcrumb trail. Use breadcrumb links to navigate back through your record history.

💡 Pro tip: Use Ctrl+F (or Cmd+F) to search within the request/response JSON. This is useful for finding specific fields in large payloads.

LMS Monitoring (Beta)

The LMS (Lightning Message Service) type pill in the component tree gives you access to message channels in your org. This feature is in beta and may not discover all channels in every org.

LMS Features

  • Channel discovery — Automatically scans the page for active LMS channels
  • Subscribe/Unsubscribe — Toggle subscriptions to monitor channel activity
  • Publish test messages — Send test payloads to a channel to verify behavior
  • Message schemas — View the structure of messages expected by each channel
  • Activity logs — See a real-time feed of all messages published to subscribed channels

LMS Status Indicator

At the top of the LMS detail pane, you'll see a status pill showing one of three states:

  • Ready (green dot) — The LMS bridge is active and channels are discoverable
  • Unavailable (gray dot) — The LMS is not available in this org
  • Not Connected (red dot) — The extension lost connection to the LMS bridge
Beta status: LMS monitoring is a new feature. Some channels may not be discoverable depending on how they're defined in your org. If you encounter channels that don't appear, try refreshing the page and clicking the Refresh Components button in the header.

Dock Mode

The LWC Explorer panel can be docked to different edges of the page to suit your workflow.

Docking Positions

  • Right side (default) — Panel docks to the right edge of the page
  • Bottom — Panel docks to the bottom of the page (useful on smaller screens)

Changing Dock Position

Look for a dock mode toggle button in the LWC Explorer header. Click it to toggle between docking positions. Your preference is automatically saved and persists across sessions.

Responsive design: On very small screens, the bottom dock position gives you more horizontal space for the component tree and trace details.

Tips & Best Practices

For Performance Debugging

  • Use Auto-Capture mode while performing an action on the page (e.g., clicking a button, opening a modal). After the action completes, click Auto-Capture again to stop recording and review the traffic that was triggered.
  • Filter by Apex to focus on server-side performance. If you see long-running Apex calls, those are the first place to optimize.
  • Look at the Timing column to identify slow requests. Anything over 500ms deserves investigation.
  • Use the Collapse All button to simplify the component tree when debugging a single component. Root nodes stay visible so you can still navigate.

For Component Inspection

  • Click the Pick Component button, then hover over any element on the page to visually select the component. This is faster than searching the tree.
  • Use the search box in the component tree to quickly navigate to a specific component by name. Matching components stay expanded.
  • When you've found the component you're interested in, the related network traces are automatically shown on the right. You don't have to click between panels.

For Data Exploration

  • Click any record ID link in a trace response to navigate to that record. Use breadcrumbs to walk back through your navigation history.
  • When reviewing UI API calls, look for the fields section in the response to see what data was returned to the component.
  • Use the Errors filter to quickly find failed API calls. Check the error message and stack trace (if available) to understand what went wrong.

For LMS Troubleshooting

  • If LMS is Unavailable, your org may not have Lightning Message Service enabled, or the discovery bridge isn't active. Try refreshing the page and clicking Refresh Components.
  • Use the publish form to test a channel with a sample message. If your subscribers receive it, the channel integration is working.
  • The activity log shows a real-time feed of messages published to channels you're subscribed to. This is invaluable for verifying that two components are communicating correctly.
Keyboard shortcuts: Use Ctrl+K (Cmd+K on Mac) to open the Command Palette from anywhere. Type "LWC" to jump to LWC Explorer, or "Clear" to clear all traces without opening the panel.

Connection Issues

Connection problems prevent TrackForcePro from communicating with your Salesforce instance. Use this table to diagnose and resolve authentication and session issues.

Issue Root Cause How to Fix
Not connected to Salesforce No Salesforce tab open Open a Salesforce tab and log in. TrackForcePro reads session from the active SF tab. Reopen the extension popup once logged in.
Session expired Salesforce session timed out Refresh the Salesforce tab to re-authenticate. Close and reopen the TFP popup to refresh the cached session.
Extension context invalidated TrackForcePro was updated while the page was open Reload the Salesforce tab. This re-initializes the content script with the latest extension code. If using Show All Data, close the panel first.
Service worker crashed MV3 service worker timed out or crashed Close and reopen the TFP popup. You may see a "degraded mode" toast notification. If persistent, restart the browser. Background.js will auto-recover on next message.
Status shows wrong org Multiple Salesforce tabs open Click into the correct Salesforce tab first. TrackForcePro reads the org from the focused tab. If using multiple orgs, keep only one SF tab active per session.

Error Code Reference

Salesforce and REST API errors are returned as HTTP status codes or Salesforce-specific error types. This reference lists the most common errors in TrackForcePro workflows.

Error What It Means How to Fix
401 Unauthorized access Session token expired or invalid. Refresh the Salesforce tab. Verify you're logged into the correct Salesforce org.
403 Forbidden — permission denied Your Salesforce user profile lacks permission for the requested action. Check with your admin to grant API permissions or object-level access (SOQL, field updates, etc.).
404 Not found Record, object, or field does not exist. Verify the record ID is correct and the object/field name matches the API name (custom fields end in __c).
400 Bad request — malformed input The request syntax is invalid. Check SOQL grammar, field names, and filter values. Use the Guidance Engine in SOQL Builder for syntax hints.
500 Internal server error Salesforce platform error. Retry the request in a few moments. If persistent, check Salesforce System Status and contact Salesforce support.
INVALID_FIELD Field does not exist on the object Verify the field API name. Check the custom field name ends with __c. Confirm the field exists on the selected object. Use the SOQL field dropdown to select valid fields.
INVALID_TYPE Field value type mismatch Value type (text, number, boolean, etc.) doesn't match field type. Example: "true" text vs boolean true. Check field type in Org Tools > Org Limits or Salesforce Setup.
MALFORMED_QUERY SOQL syntax error Check SOQL grammar: parentheses balanced, clauses in order (SELECT … FROM … WHERE … ORDER BY …). Use Guidance Engine to auto-correct common mistakes.
DUPLICATE_VALUE Unique field constraint violated Field marked "Unique" already has this value on another record. Check existing records using SOQL or Org Tools. Update or delete conflicting record first.
STRING_TOO_LONG Text exceeds field length Text field has a character limit. Check Org Tools for field max length. Shorten the value or request field length increase from admin.
REQUIRED_FIELD_MISSING Mandatory field is empty Required fields must have a value. For inline edits, ensure all required fields on the record are populated. Check Org Tools for required field list.
NetworkError Network connectivity lost Check internet connection. Verify firewall/proxy allows Salesforce API. Browser console may show "Failed to fetch." Retry the request.
Request timed out API response exceeded 30 seconds Query is too complex or processing too many records. Reduce result set: add WHERE filters, increase LIMIT, or split into smaller queries. Check SOQL performance.

SOQL & Query Issues

Query-related problems occur in SOQL Builder, GraphQL Explorer, and REST Explorer. Symptoms include missing objects, invalid fields, empty results, or disabled editing.

Issue Root Cause How to Fix
No objects in dropdown Session not connected Ensure you're logged into Salesforce and the tab is open. Click into the Salesforce tab, then reopen TFP popup. Session is fetched on popup load.
INVALID_FIELD error Field doesn't exist on object Verify field API name (copy from Setup). Custom fields end with __c. Use the SOQL field dropdown to auto-suggest valid fields. Check field-level security.
MALFORMED_QUERY error SOQL syntax error Check Guidance Engine hints (red indicators). Ensure parentheses are balanced. Verify clause order: SELECT, FROM, WHERE, GROUP BY, ORDER BY, LIMIT. Test simple query first.
Results are empty No matching records Remove or broaden WHERE filters. Start with a simple query: "SELECT Id FROM Account LIMIT 10". Check record count in Salesforce UI. Verify filter syntax and field types.
Inline edit doesn't work Inline editing disabled for this org Check Settings > Org Management. Verify "Allow Editing" toggle is enabled for your org. If disabled, click to enable. Changes apply immediately to new edits.
API limit exceeded Org hit daily REST API call limit Check Org Tools > Org Limits for remaining API calls. Reduce query frequency. Wait until next day for limit reset (typically 12:00 AM UTC). Contact admin for limit increase.

Performance Issues

Slow queries, laggy UI, or buffer overflows may occur with large datasets or complex operations. These solutions optimize resource usage.

Issue Root Cause How to Fix
Query times out Too many records or slow filters Add LIMIT clause: LIMIT 5000. Add WHERE filters to narrow results. Use indexed fields in WHERE clauses. Avoid complex calculations in SELECT clause.
Virtual table laggy Too many columns (50+) or rows Use Smart View to auto-hide empty/null columns. Uncheck columns not needed. Select a higher density (Compact or Dense) for smaller rows. Reduce result LIMIT.
LWC Explorer buffer full Auto-Capture collected too many network traces Click "Clear All" to reset buffer. Buffer caps at 200 traces. Turn off Auto-Capture if not needed. Traces older than ~10 minutes auto-expire.
Bulk scan slow Processing 200+ records concurrently Bulk scan is rate-limited to 5 concurrent API calls. Process 200 IDs max per scan. Wait for completion progress bar (green). Split large scans into multiple batches.
Settings won't save Browser storage quota exceeded Clear Query History: Settings > Backup > Clear Query History. Export and delete old backups. Restart browser. If persistent, clear site data for extension domain.

Cross-Browser Issues

TrackForcePro runs on Chrome, Edge, Firefox, and Opera. Some features behave differently per browser due to API limitations.

Browser Issue Resolution
Firefox declarativeContent not supported This API is Chrome/Edge-only. TFP automatically falls back to manual context menu triggers. No action needed — fully supported.
Firefox Popup size differs from other browsers Use the Pop-out button (opens results in a new tab) for full-width view. Firefox popup window size is controlled by the browser and cannot be customized by extensions.
Edge Updates lag behind Chrome release Check edge://extensions and click "Update" manually. Enable auto-update in Edge settings (enabled by default). Allow 24-48 hours for automatic propagation.
Opera chrome.commands may not register Use the sidebar toggle button or keyboard shortcut via custom settings. chrome.commands support in Opera is limited. Context menu and manual shortcuts always work.
Safari Not supported Safari does not support Manifest V3 extensions. Use Chrome, Edge, Firefox, or Opera instead. No Safari port is planned at this time.

LWC Explorer Issues

LWC Explorer is a page-side inspector for Lightning Web Components, Aura, and LMS. Issues here relate to component detection and network tracing.

Issue Root Cause How to Fix
"Not connected" LMS status Page doesn't have LMS or is non-Lightning LMS only works on Lightning pages with the messaging service. This is normal for non-Lightning Salesforce pages. No action needed.
Component tree empty No components detected on page Refresh the Salesforce page. Ensure you're on a Lightning page with components (Home, App Builder, etc.). Some pages may not have explorable components.
Auto-Capture not capturing Extension context invalid Reload the Salesforce page. This re-initializes the network injected script. If TFP was updated, you must reload the SF page to load new code.
Traces not showing Filter is active (not "All") Click the "All" filter chip at the top to show all trace types. Click specific chips (Apex, UI API, GraphQL, etc.) to filter by type. Traces matching the filter will appear.

Frequently Asked Questions

Does TrackForcePro store my credentials? +

No. TrackForcePro never stores or transmits your Salesforce username, password, or security token. It reads your session token from the browser's cookies (automatically set by Salesforce) and uses it to make API calls. Session tokens are browser-managed and never persisted to extension storage.

Can TrackForcePro access my data when the extension is closed? +

No. TrackForcePro only runs while the popup or content scripts are active. When you close the popup, the extension stops. It cannot access data, read pages, or make API calls in the background. The service worker (background.js) only handles message routing — it does not make unsolicited API calls.

Does inline editing work on production orgs? +

Yes, inline editing is enabled by default and works on all org types (dev, sandbox, production). You can disable it per-org in Settings > Org Management if needed. Admins can toggle "Allow Editing" for each org individually. Inline edits use the Salesforce REST API PATCH endpoint with full validation.

Why don't I see all Salesforce objects in SOQL Builder? +

Your Salesforce user profile determines which objects are visible. If you cannot see an object, your profile likely lacks "Read" permission for that object. Contact your Salesforce admin to grant API access to the object. Custom objects are visible only if your profile has access. The object list is filtered by your actual Salesforce permissions.

Can I use TrackForcePro with multiple orgs simultaneously? +

Yes. TrackForcePro uses org-scoped caching, which prevents data leaks between orgs. You can have multiple Salesforce tabs open, each logged into a different org. TrackForcePro reads the session from the currently focused tab. To avoid confusion, keep only one org tab active at a time, or use multiple browser windows.

What Salesforce API version does TrackForcePro use? +

TrackForcePro uses Salesforce API versions 60–66, automatically detected from your org. The API version is determined by your Salesforce instance and org settings. In Org Tools, you can see your current instance URL and API version. No manual version selection is needed.

How do I reset TrackForcePro to factory defaults? +

Go to Settings > Backup > Reset All. This clears all settings, cache, history, and saved preferences. Confirm the action (it cannot be undone). Your Salesforce data is not affected. After reset, all defaults are restored. You can also export settings before reset as a backup.

Can I export and share my TrackForcePro settings? +

Yes. Go to Settings > Backup > Export All to download a JSON file with all your settings, including theme, layout, keyboard shortcuts, and saved orgs. You can share this file with teammates. To import, click Import All and select the exported JSON file. Settings merge with existing settings (imports do not overwrite).

Is my Salesforce data sent outside my browser? +

No. TrackForcePro runs entirely in your browser. All API calls go directly from your browser to your Salesforce instance. No data is sent to TrackForcePro servers or external services. The extension is open-source and makes no outbound network calls except to Salesforce APIs. Query results and records are cached locally in your browser storage.

Why does LWC Explorer show a "Beta" badge? +

LMS (Lightning Message Service) is in beta and may change in future releases. The rest of LWC Explorer (component inspection, network tracing, Aura/GraphQL capture) is stable. The beta badge indicates that the LMS UI and APIs may be refined. Feedback is welcome. All other features in TrackForcePro are production-ready.

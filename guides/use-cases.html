Learn by Doing

⚡ This page is loading slowly

Developer debugging a slow Lightning record page

Debugging with LWC Explorer — capture network traffic, identify slow Apex calls, and optimize performance.

Open slow Lightning record page in Salesforce
Click "LWC Explorer" from the TrackForcePro sidebar
Enable "Auto-Capture" to start recording all network traffic
Reload the page and watch the network activity populate in real-time
Filter by "Apex" to isolate server-side calls
Sort traces by duration (longest first) to find bottlenecks
Click the slowest trace to inspect the class name, method, and response time
✓ Result:

Found AccountController.getRelatedData takes 4.2 seconds due to unoptimized SOQL with missing indexes. Optimized query and page now loads in 600ms.

LWC Explorer Auto-Capture Apex Filter
💡 Tip: Filter by "Errors" to check for failed API calls and retries that artificially inflate page load time.

🔐 Who changed the permission set?

Admin investigating an unexpected permission change

Audit Trails investigation — search security changes and find the exact user and timestamp.

Open Audit Trails tab in TrackForcePro popup
Click "Fetch" to load audit entries from your org
Search for the permission set name (e.g., "Account Manager")
Filter "Category" to "Security" to show only permission-related changes
Expand entries to view the user, date/time, and action (e.g., "Added" or "Removed")
Export results as CSV to include in incident report or compliance documentation
✓ Result:

Discovered junior admin removed "View All Data" permission set yesterday at 3:47 PM. Contacted user to understand reason and reapplied if needed.

Audit Trails Search Category Filter Export CSV

📊 Export all contacts for quarterly report

Admin preparing data export for reporting

SOQL Builder — craft a query, filter by date, and export records in seconds.

Open SOQL Builder tab
Select "Contact" from the object picker
Use the field picker to select columns: Name, Email, Phone, LastActivityDate
Add a WHERE clause to filter: CreatedDate = THIS_QUARTER
Add ORDER BY LastName ASC to sort alphabetically
Click "Run Query" to execute
Click "Export CSV" and save to your computer
✓ Result:

Exported 2,347 contacts in under 60 seconds. Imported CSV into Excel, removed duplicates, and delivered to analytics team.

SOQL Builder Field Picker Filter Export CSV

🔒 Check why a user can't see a field

Admin troubleshooting field-level security (FLS)

Security Manager — analyze field permissions and profile assignments instantly.

Open Records tab and enter the record ID in the search field
Navigate to "All Fields" section to see all record fields
Search for the field name (e.g., "Revenue__c") in the field list
Click "Security Manager" sub-tab to analyze permissions
Review FLS analysis showing which profiles have read/write access
Check the affected user's profile in the analysis (e.g., "Standard User")
Alternatively, use "Login As" in Org Tools to verify permissions firsthand
✓ Result:

Confirmed that user's "Standard User" profile has "No Access" to Revenue__c field. Updated profile FLS, user can now view field.

Records Tab Security Manager FLS Analysis Org Tools Login As
💡 Tip: Use "Login As" to impersonate the user and verify exactly what they can/cannot see without affecting their session.

📡 Monitor integration events in real-time

Integration developer testing event-based integrations

Platform Events — subscribe to custom events and inspect payloads as they publish.

Open Platform Events tab in TrackForcePro popup
Click "Fetch" to load all available platform events from the org
Find and click your custom event (e.g., "Order_Placed__e")
Click "Subscribe" to start listening for new events
Trigger your integration workflow in Salesforce (e.g., create a test order)
Watch the event log populate in real-time with incoming events
Expand and inspect the event payload to verify field types and data
✓ Result:

Confirmed that Order_Placed__e events publish correctly, but discovered Amount field is sent as a string ("1500.00") instead of a number. Updated trigger to cast field correctly before publishing.

Platform Events Subscribe Payload Inspection
💡 Tip: Keep Platform Events open in a separate popup window while testing your integration to monitor event flow in real-time.

🚀 Test this API endpoint before coding

Developer prototyping API integration

REST Explorer — prototype CRUD operations and build reusable collections.

Open REST Explorer tab
Select "POST" method and change the resource path to "/sobjects/Account"
Paste a JSON body with new account data (Name, BillingCity, Industry, etc.)
Click "Send Request" and verify a 201 Created response with the new ID
Change method to "PATCH" with the ID and update a field (e.g., Revenue)
Verify the PATCH succeeds with a 204 No Content response
Test DELETE with the same ID to confirm record removal
Save this sequence to a collection for future reference
✓ Result:

Prototyped full CRUD cycle in under 3 minutes. Saved as reusable collection "Account_API_Tests" for integration development.

REST Explorer Collections Request Builder

🌐 Deploy custom labels to a new locale

Admin deploying multi-language labels

Org Tools — export, translate, and batch-import custom labels across languages.

Open Org Tools tab → Custom Labels panel
Export all labels as a translation template (CSV format)
Send the CSV to your translation vendor or use Google Translate API
Receive translated CSV with Spanish, French, German, or other locales
Use the import feature to validate and parse the translated file
Review the parsed label list for correct language mapping
Click "Apply" to batch-create all 150+ labels in Salesforce
✓ Result:

Deployed 150 Spanish translations in one batch operation. All labels published to production and instantly available across the org.

Org Tools Custom Labels Import/Export
💡 Tip: Always test custom labels in a non-production org first to verify translation quality and formatting before deploying to production.

🐛 Debug an Apex trigger not firing

Developer debugging trigger logic

Debug Logs + LWC Explorer — capture execution flow and identify filtering issues.

Open Org Tools tab → Debug Logs panel
Create a new Trace Flag for your user with DEBUG log level
Open the Salesforce record page in a new tab alongside LWC Explorer
In LWC Explorer, enable "Auto-Capture" to monitor all network traffic
Perform the action that should trigger the trigger (e.g., update a field)
Filter LWC Explorer by "Apex" to see the trigger execution trace
Check the captured debug log for entry/exit logs and error messages
Review the trace to identify where execution stops or unexpected logic branches occur
✓ Result:

Debug logs revealed sharing rule filtered record before trigger could execute. Updated sharing rule, trigger now fires correctly on all qualifying records.

Org Tools Debug Logs LWC Explorer Apex Filter Auto-Capture
💡 Tip: Always check Sharing Rules and Org-wide defaults before debugging trigger logic — they can silently filter records before your code runs.

⚙️ Set up TrackForcePro for a team

Lead admin configuring TrackForcePro standards

Settings — appearance, shortcuts, and security configuration for team consistency.

Open Settings tab (⚙️ icon) in TrackForcePro popup
Navigate to "Appearance" sub-tab and choose your preferred theme (Light/Dark/Auto)
Go to "Keyboard Shortcuts" sub-tab and configure custom shortcuts for common tasks
Navigate to "General" sub-tab and disable inline editing for production orgs
Open "Sidebar" sub-tab to hide unused features and customize button positions
Review "Org Management" to see connected orgs and toggle edit access per-org
Click "Save & Apply" to apply all settings
Export all settings as a JSON backup (coming in v2.0)
Share backup file with team members to import identical configuration
✓ Result:

Entire team configured consistently with dark theme, disabled prod editing, and shared keyboard shortcuts. Reduced setup time from 30 min per person to 2 min via configuration sharing.

Settings Tab Appearance Keyboard Shortcuts Org Management Sidebar Configuration
💡 Tip: Always disable inline editing on production orgs to prevent accidental data modifications. Keep it enabled only in sandboxes for testing.

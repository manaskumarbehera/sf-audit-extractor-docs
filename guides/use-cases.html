Debug Why a Salesforce Page Loads Slowly

Your users complain that a record page is slow. Find the culprit in minutes.

Workflow

  1. Open the record in Salesforce
  2. Open LWC Explorer (keyboard shortcut or sidebar)
  3. Select "Live Mode" to stream network traffic
  4. Look for slow Apex or UI API calls — Filter by "Apex" or "UI API" chips
  5. Identify the bottleneck — Is an Apex REST call taking 3+ seconds? Is a UI API query fetching 10k records?
  6. Optimize or escalate — If it's Apex, work with your dev team to optimize the code. If it's config, investigate field security or record access logic.
Pro Tip: Check the ChangeEventHeader in CDC events. If many records are being modified simultaneously, check if a batch job or integration is running.

Investigate Who Changed What and When

Someone changed a critical field. Track them down using Audit Trail + Show All Data.

Workflow

  1. Open Audit Trail from sidebar or workspace
  2. Filter by date range (narrow it to the time the change happened)
  3. Filter by section (e.g., "Users", "Security", "Organization")
  4. Find the relevant entry and note the user and timestamp
  5. If you need field history, open the record and click "Field History" in the sidebar
  6. Export the results and attach to your compliance audit trail

Bulk Export Specific Records for Analysis

Export 1000s of records to a spreadsheet for external analysis, without navigating Salesforce UI.

Workflow

  1. Open SOQL Builder
  2. Write a SOQL query to select the records you want (e.g., "SELECT Id, Name, Email FROM Contact WHERE Industry = 'Technology'")
  3. Execute the query
  4. In the results toolbar, click Export → CSV Download
  5. Open the CSV in Excel, Google Sheets, or your analysis tool
Faster than Data Export: The standard Salesforce Data Export tool can take hours for large orgs. TrackForcePro's SOQL export is instant.

Investigate Record Sharing & Access Issues

A user can't see a record they should have access to. Debug the sharing rules.

Workflow

  1. Open the record in Salesforce
  2. Click "Sharing" in the sidebar
  3. Review who has access — Check if the user is in the list, their access level (View Only vs. Edit), and through what mechanism (direct share, public group, role hierarchy)
  4. If access is missing, check:
    • Is the user's role higher in the hierarchy than the record owner?
    • Is the user in a public group that should have access?
    • Are there org-wide defaults limiting access?
  5. Add access manually or fix the rules in Salesforce Setup

Test Salesforce API Integrations Before Deployment

You're integrating with a third-party API. Test the REST endpoints quickly.

Workflow

  1. Open REST Explorer
  2. Find the Salesforce API docs for the endpoint you need (e.g., Leads, Accounts)
  3. In REST Explorer, enter the method (GET, POST, PATCH)
  4. Enter the endpoint URL (e.g., `/services/data/v60.0/sobjects/Account/create`)
  5. Enter request body (JSON with fields to create/update)
  6. Execute and check response — If you get 201 Created, your integration logic is correct. If 400, debug the request format.
  7. Once working, pass this REST Explorer request to your dev team as documentation of what works

Monitor Org Limits Before Large Operations

You're about to run a bulk data load. Make sure you have capacity.

Workflow

  1. Open Org Tools → Org Limits
  2. Check API Calls remaining — If you're at 90% of your daily limit, pause and wait for the reset
  3. Check Data Storage — If you're at 95%, you might not have room for bulk inserts
  4. Check File Storage — If bulk operation involves attachments, verify space
  5. If limits are tight, split the operation into smaller chunks spread over multiple days
Best Practice: Check limits before starting ANY large operation (bulk update, batch job, integration sync). Takes 10 seconds and prevents hours of troubleshooting.

Compare Field Values Across Multiple Records

Three accounts look similar. Find the differences.

Workflow

  1. Open Show All Data on the first record
  2. Search for the field you want to compare (e.g., "BillingCity")
  3. Copy the value (click to copy)
  4. Navigate to the second record using the breadcrumb or search
  5. Show All Data opens on the new record
  6. Search for the same field and compare values
  7. Repeat for the third record
Alternative: Use SOQL Builder to query all three records in one go, then compare in the results table.

Troubleshoot Formula Field Logic

A formula field shows unexpected results. Debug it.

Workflow

  1. Open Show All Data on the record with the formula field
  2. Search for the formula field (e.g., "Total_Price__c")
  3. See the current value — Is it what you expect?
  4. Check the component fields that feed the formula (e.g., if Total = Price × Quantity, check Price and Quantity values)
  5. If values are wrong, check:
    • Did a workflow/flow recently update these fields?
    • Are there rounding or data type issues in the formula?
    • Is the formula referencing a lookup field that might be NULL?
  6. View the formula definition in Salesforce Setup (Setup → Object Fields → Formula Field) and run through the logic manually

📋 Audit Trail Guide

What It Does

The Audit Trail tab lets you view and export Salesforce Setup Audit Trail entries directly from TrackForcePro. Quickly filter by user, action type, date range, and section. Export results to CSV for compliance reporting, change tracking, and audit investigations.

This is essential for admins who need to investigate who changed what and when, without navigating deep into Salesforce Setup.

How to Access

Open the TrackForcePro workspace (click the extension icon) and click the Audit tab — it's the first tab in the workspace. The tab automatically loads recent audit trail entries from your connected org.

Once loaded, you'll see a filter bar at the top and a results table below showing all available audit trail entries.

Filtering

Use the filter bar to narrow audit trail results. All filters are optional and combine with each other:

  • Date Range Picker — Click the date fields to select a start date and end date. Only entries within this range will be shown.
  • User Filter — Search by username or email. The filter shows matching users as you type.
  • Action Filter — Select specific action types (e.g., "Created", "Updated", "Deleted") from the dropdown.
  • Section Filter — Choose setup sections like "Security", "Users", "Organization", "Data & Monitoring", etc. Only entries from that section will appear.

You can use multiple filters at once. For example: filter by date range AND user AND action to see exactly what one user changed on a specific date.

Results Table

The results table displays audit trail entries in columns:

Column Description
Date/Time When the change was made (timestamp).
User The Salesforce user who made the change.
Action The specific action (e.g., "User Updated", "Profile Modified", "Permission Set Created").
Section The setup area where the change occurred (e.g., "Users", "Security").
Delegate User If applicable, the user on whose behalf the change was made.

Click any column header to sort in ascending or descending order. Results load in pages for performance — use the pagination controls to navigate.

Export

Click the Export button to download filtered results as a CSV file. The export respects all your current filters — only matching entries are included in the file.

The CSV can be opened in Excel, Google Sheets, or any spreadsheet application for further analysis, pivot tables, or sharing with auditors.

Tips & Best Practices

  • Narrow Large Audit Trails — If your org has years of audit data, always use a date range filter to keep results manageable.
  • Export for Compliance — Export filtered results and save them as evidence for compliance audits or regulatory reviews.
  • Combine Filters — Use user + action + date filters together to investigate specific changes quickly. For example: "What did Alice do to Security settings last month?"
  • Delegate User Tracking — If you see entries with a Delegate User, those are actions taken on behalf of another user (common for IT support scenarios).
  • Regular Reviews — Schedule weekly or monthly exports to keep a record of admin activity for governance.
💡 Pro Tip: If an unexpected change happened, filter by the affected section and a narrow date range. Then export and cross-reference the user against your access logs.

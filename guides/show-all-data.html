📊 Show All Data Guide

View all fields, values, and metadata for any Salesforce record directly on the page.

In-Page Overlay

What It Does

Show All Data opens a comprehensive inspector panel for any Salesforce record, displaying every field with its current value, API name, data type, and editability status. Instead of hunting through Setup, navigating multiple screens, or using Data Loader to inspect record data, you get instant visibility to all fields right on the record page.

The floating button appears on every Salesforce record detail page, making data exploration immediate and contextual. No page reloads, no navigation delays — just click and explore.

How to Open

You have three ways to launch Show All Data:

  1. Floating Button: Look for the "TFP Show All Data" button that floats on the right side of the record page (or left/bottom if you've configured it). Click it to slide open the side panel.
  2. Right-Click Menu: Right-click anywhere on a Salesforce record page and select "Show All Data" from the TrackForcePro context menu.
  3. Keyboard Shortcut: Press the configured keyboard shortcut (default varies, configurable in Settings → Sidebar). The panel slides in instantly without any mouse interaction.

Once open, the panel slides in from the right (or configured side), overlaying the record details. Click the close button or press Escape to dismiss it. The sidebar stays independent of your Salesforce browsing — it won't interfere with form submission or page navigation.

Field Explorer

The Show All Data panel displays a comprehensive, paginated table of all fields on the record. Each page shows up to 100 fields to keep the interface responsive.

Column Description
Field Label The user-friendly name as shown in Salesforce UI (e.g., "Account Name").
API Name The technical API identifier used in queries and integrations (e.g., "Name"). Click to copy to clipboard.
Value The current value stored in the field. Shows "— " if empty or null.
Type The field's data type: Text, Picklist, Lookup, Formula, Currency, Boolean, Date, DateTime, etc.

At the top of the panel is a search bar. Type any part of a field label or API name to filter the table in real time. This is especially useful when working with objects that have 50+ fields — search for "phone" to find all phone-related fields instantly.

Fields are color-coded by type for quick visual scanning. Lookup fields show a clickable link (see Lookup Navigation below).

Inline Editing

TrackForcePro lets you edit field values directly from the Show All Data panel without opening edit mode or a separate form. Double-click any updateable field to start editing.

The input type changes based on the field's data type:

  • Text / Email / URL: Text input box. Type your value and press Enter to save.
  • Boolean: Dropdown with True/False options. Select and press Enter to save.
  • Long Text / Rich Text: Textarea. Press Ctrl+Enter (or Cmd+Enter on Mac) to save, or click outside to cancel.
  • Picklist: Dropdown with available values. Select and press Enter to save.
  • Date / DateTime: Date picker. Select the date and time, then press Enter to save.

Press Escape at any time to cancel editing without saving. After you press Enter, TrackForcePro sends a PATCH request to the Salesforce REST API to update the field. A success toast confirms the change.

Important: Inline editing can be disabled per-org in Settings → General → Org Management. If inline edit is blocked for your org, the double-click action is disabled and you'll see a warning notification.

Lookup Navigation

When a field contains a lookup to another Salesforce record (Account, Contact, Opportunity, etc.), the value appears as a clickable link in the Show All Data table. Click the link to navigate to that related record.

As you click through related records, TrackForcePro builds a breadcrumb trail at the top of the panel. For example: "Account (001xx) > Contact (003xx) > Opportunity (006xx)". This trail lets you see your navigation history without getting lost.

Click any breadcrumb to jump back to a previous record. The breadcrumb list truncates as you navigate deeper, keeping the UI clean.

This feature is perfect for exploring related data structures: start with a record, follow its parent Account, check the Contact, then jump back to the original Opportunity — all without leaving the Show All Data panel.

Related Actions from Context Menu

The right-click context menu on Salesforce record pages includes two additional Show All Data–related shortcuts:

  • Field History: Opens a history log of changes to tracked fields on the current record. Shows when each field was changed, who changed it, and the old/new values (if field history tracking is enabled on your org).
  • Sharing: Displays the current record's sharing rules and access levels. See who has what level of access (Read, Edit, All) to this record.
  • Copy Record ID: Copies the record's 18-character ID to your clipboard for quick reference in SOQL queries or API calls.

Tips & Tricks

💡 Tip 1: Search by API Name
If you know the technical API name of a field (e.g., "CustomField__c"), search for it directly instead of scrolling. The search bar filters on both label and API name.
💡 Tip 2: Production Orgs Are Safe
Inline editing is automatically disabled for any org marked as Production in Settings. This prevents accidental field updates on live data. To edit production data, you must explicitly enable it per-org.
💡 Tip 3: Explore Record Networks
Use lookup navigation to explore your record relationships. Start with one record, click related lookups, and use breadcrumbs to backtrack. It's a visual way to understand data structure without writing a SOQL query.
💡 Tip 4: Copy API Names Fast
Click the API Name in the table to copy it to your clipboard. Perfect for building SOQL queries or referencing fields in integrations.
💡 Tip 5: Keyboard Shortcut Efficiency
Set a keyboard shortcut for Show All Data in Settings → Sidebar. Pressing that shortcut is faster than clicking a button or using the right-click menu, especially if you inspect records frequently.
💡 Tip 6: Position Independence
The Show All Data button position is independent from the sidebar and Command Palette. If your sidebar is on the left, put the Show All Data button on the right. Configure all three independently in Settings → Sidebar.

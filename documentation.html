⬇️ Installation

Chrome Web Store (Recommended)

Visit the Chrome Web Store and search for TrackForcePro, or use the direct link to install the extension.

  1. Open Chrome and navigate to the Chrome Web Store
  2. Search for "TrackForcePro"
  3. Click "Add to Chrome"
  4. Confirm the permissions prompt
  5. The extension icon will appear in your toolbar

Manual Installation (Developer Mode)

For development or when the extension is not yet available in the Web Store:

  1. Download the extension files from GitHub
  2. Extract the ZIP file to a folder
  3. Open Chrome and go to chrome://extensions/
  4. Enable "Developer mode" (top-right toggle)
  5. Click "Load unpacked"
  6. Select the extension folder
  7. TrackForcePro will appear in your extensions list
💡
Browser Requirements TrackForcePro requires Chrome 90+ or any Chromium-based browser (Edge, Brave, etc.)

🚀 Quick Start

First Launch

  1. Install TrackForcePro from the Chrome Web Store
  2. Navigate to your Salesforce org
  3. Click the TrackForcePro icon in your toolbar
  4. The popup will appear with quick links to all features

Common Tasks

  • Extract Audit Trails - Click "Audit Extractor" and configure your filters
  • Query with SOQL - Use the SOQL Builder for visual query composition
  • Scan Records - Navigate to any record and click "Record Scanner"
  • Manage Users - Go to Setup > Users to access User Manager
  • Set Org Favicon - Click the extension icon and select "Favicon Manager"

Badge Labels

Up to 3 characters displayed on the favicon icon (e.g., DEV, PRD, QA, UAT). Labels are rendered in bold white text with adaptive outlines for readability on both light and dark icon colors.

How Favicons Work

  • Creation — A 32×32 canvas is drawn with the selected shape, color, and label. The canvas is converted to a PNG data URL.
  • Storage — Favicon settings are saved to chrome.storage.local in two structures: orgFavicons (legacy, keyed by org ID) and orgRecords (comprehensive org data with iconConfig).
  • Rendering — The drawFaviconShape() function exists in both content.js and data_explorer_helper.js to ensure preview matches the actual favicon.
  • Application — On page load, content.js calls applyFaviconOnLoad() which reads saved favicons from storage and matches by org ID (exact or normalized 15-char), hostname, or partial hostname.
  • Tab Application — When saving, applyFaviconToTabs() sends updateFavicon messages to all matching Salesforce tabs (filtered by hostname to prevent cross-org contamination).

Org Branding Features

  • Logo Colorization — The Salesforce cloud logo in the page header is tinted to match your org color using CSS hue-rotate() filters. A colored bar indicator also appears below the logo.
  • Tab Title Customization — The browser tab title is prefixed with the badge label (e.g., [DEV] Account | Salesforce).
  • Multi-Org Support — Each org stores its own favicon settings independently. Favicons auto-apply based on org ID detection from the page.
  • Per-Org Isolation — Favicon updates include org ID verification to prevent cross-org contamination when multiple orgs are open simultaneously.

🔧 Troubleshooting

Extension not appearing in toolbar

Solution: Check that the extension is enabled in chrome://extensions/. If installed from Chrome Web Store, wait a few moments for it to activate.

Favicon not applying to tab

Solution: Ensure you're on a Salesforce page (*.salesforce.com). Clear the extension cache and refresh the page.

SOQL query not executing

Solution: Verify that your org has API access enabled and you're not hitting rate limits. Check the browser console for error messages.

Record Scanner showing no data

Solution: Confirm you have FLS permissions to view the record. Try reloading the page and launching the scanner again.

Permission errors when using tools

Solution: Ensure your Salesforce user has sufficient permissions for the operation. Some tools require admin access.

Cache or performance issues

Solution: Open Cache Manager and clear all cache. Then refresh your Salesforce tab. If the problem persists, try uninstalling and reinstalling the extension.

📚 API Reference

Extension API

TrackForcePro provides several APIs for developers who want to extend or integrate with the extension.

Message Passing

The extension uses Chrome's message passing API for communication between content scripts and background scripts.

chrome.runtime.sendMessage({
  action: 'queryRecords',
  query: 'SELECT Id, Name FROM Account LIMIT 10'
}, function(response) {
  console.log(response.records);
});

Storage API

Configuration and cached data are stored using chrome.storage.local and chrome.storage.sync.

chrome.storage.local.get(['orgFavicons'], function(result) {
  console.log('Saved favicons:', result.orgFavicons);
});
